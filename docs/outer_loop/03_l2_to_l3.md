# 3. L2 → L3 스킬 결정

## 책임

검증된 L2 지식 중 반복 가능한 행동 순서를 가진 패턴을 Seed 스킬로 만들고,
기존 스킬의 운영 성과로 생명 주기를 갱신한다.

## 의사 코드

```text
function manage_procedural_memory(changed_l2, skill_policy):
    for knowledge in changed_l2:
        if is_repeatable_procedure(knowledge) and has_sufficient_evidence(knowledge):
            skill = find_skill_by_knowledge(knowledge.id)
            if skill is null:
                create_seed_skill(
                    source_l2_id = knowledge.id,
                    procedure = derive_steps(knowledge),
                    operating_conditions = knowledge.conditions
                )
            else:
                revise_seed_skill(skill, knowledge)

    for skill in list_non_archived_skills():
        performance = aggregate_skill_operations(skill, skill_policy.window)
        transition = decide_skill_lifecycle(
            current_status = skill.status,
            success_rate = performance.success_rate,
            min_samples = skill_policy.min_samples,
            active_threshold = 0.90,
            degradation_threshold = skill_policy.degradation_threshold
        )
        apply_skill_transition(skill, transition)

    return skill_changes
```

## 규칙

- L3 Seed는 “일반 지식”이 아니라 실제 실행 가능한 절차여야 한다.
- Active 승격에는 운영 표본 수와 성공률 0.90 이상이 모두 필요하다.
- L3 Active와 에이전트의 L4 자율성 단계는 서로 다른 개념이다.

# 2. L1 → L2 일반화 결정

## 책임

누적된 Pattern Candidate를 검증해 L2 조건부 지식으로 승격하거나, 기존 L2를
수정·약화·폐기한다. L2는 사례의 요약이 아니라 근거가 연결된 일반화다.

## 의사 코드

```text
function decide_l1_to_l2(changed_candidates, policy):
    changes = []

    for candidate in changed_candidates:
        evidence = load_candidate_evidence(candidate)
        recent_validity = evaluate_recent_validity(candidate, policy.health_window)
        decision = evaluate_generalization(
            support_count = evidence.support_count,
            counterexample_count = evidence.counterexample_count,
            condition_diversity = evidence.condition_diversity,
            success_rate = evidence.success_rate,
            min_cib_score = evidence.min_cib_score,
            recent_validity = recent_validity
        )

        if decision == "promote":
            upsert_l2_knowledge(
                statement = make_conditional_statement(candidate),
                conditions = candidate.conditions,
                confidence = calculate_confidence(candidate),
                supporting_episode_ids = candidate.support_ids,
                counterexample_episode_ids = candidate.counterexample_ids
            )
            candidate.state = "promoted"

        else if decision == "refine":
            narrow_candidate_conditions(candidate)
            revise_linked_l2_knowledge(candidate)

        else if decision == "weaken":
            lower_l2_confidence(candidate)

        else if decision == "retire":
            retire_l2_knowledge(candidate.linked_l2_id)

        save(candidate)
        changes.append(decision)

    return changes
```

## 결정 기준

- 충분한 누적 지지와 서로 다른 사례의 재현성이 있어야 한다.
- CIB를 통과한 증거만 일반화 근거가 된다.
- 반례는 지식을 즉시 삭제하기보다 조건을 좁히는 근거가 된다.
- 최근 N개는 과거 증거를 지우지 않지만, 지식이 아직 유효한지 확인한다.
